NovaSoda

A small web application with two cans and a glass bottle you can drag around in the browser.

Three Products

There are two cans (Citrus and Berry) and one glass bottle (Nova Cola). I built all three models using Three.js by revolving a 2-dimensional outline around its vertical axis, creating something called a lathe mesh.

Interactive

You have the ability to drag to rotate, scroll to zoom, change the product from the list on the left, switch the lights on and off, select between three different label designs, and click on the model to make it spin.

Web tech

I used HTML5, CSS3, Bootstrap 5 and jQuery for the front end. The 3D side of the project was built using Three.js r160, my own GLSL shader for creating the cola bottle glass, and a bloom pass to make the image look better. I wrote a small form that writes feedback to a SQLite database on the About page using a small PHP script.

About this 3D App

NovaSoda is a fictitious brand of soft drink. I created it to use as the basis of my H7006 Web 3D Applications coursework, because the brief specified that we could not reuse any Coca-Cola branding from the labs. The website is a simple way for me to demonstrate that I can put 3D models into a standard web page, using the tools and techniques we have learned in the course.

How I built the 3D models

I did not import pre-existing 3D files for any of the products. I entered a 2D outline for each product into the code as a list of points, and then asked Three.js to spin that outline around a vertical axis (with LatheGeometry) to create the three-dimensional product. The two cans contain approximately 600 triangles each, while the bottle contains about 1,800 triangles; and I believe this number is low enough to keep the page running smoothly on my laptop's integrated GPU.

None of the labels used image files. When the 3D app page loads, the product's brand name and flavour name are drawn to a Canvas object in JavaScript, and then the Canvas is passed to Three.js as a CanvasTexture. The texture is then used as a cover for a thin cylinder that is positioned just inside the side of each can. By clicking on either the Standard, Night or Retro 80s label, the brand and flavour name will be redrawn to the same Canvas with a different palette, and the texture will change dynamically, so that three separate PNGs were not required.

The Nova Cola bottle is the most complicated product in the project. It has a secondary, smaller mesh inside it that represents the beverage (liquid) inside the bottle, while the glass of the bottle is created using a ShaderMaterial with my own vertex and fragment shader. The colour values used for drawing the bottle's glass use a Fresnel factor, which makes the colour more intense at extreme viewing angles, and a time-based shimmer (at a slow rate) on the rim, so the outline of the bottle reads as solid amber glass, while the liquid mesh is clearly visible in the middle.

Design choices

Because the bloom and amber shader will be lost on a white background, I used a dark background for the 3D application. I initially tried a white background, but the bottle appeared flat.

The 3D app is laid out in a grid style (Bootstrap's 12 column grid). The gallery takes up 2 columns, the viewer takes up 7, and the controls take up 3. On a phone, each of the three columns will stack vertically.

The scene is set up using a three-light system (key, fill, rim). Each of the three lights is controlled via a toggle switch on the user interface. I found it easier to diagnose what each light was doing during the development phase, and I kept the switches available in case that may help a visitor understand how the lighting is working.

Navigation does not load new HTML files, but instead uses jQuery to hide/show sections on the page. This is closer to what a single-page application (SPA) does, and it is how the content swapping mentioned in the project brief is implemented here.

Accessibility

The same items that can be clicked on the 3D app page also have keyboard shortcuts. You can press the 1, 2, or 3 key to switch products; W will toggle between wireframe view or regular view; R will allow for the model to automatically rotate; and S will make the model spin once.

I did not remove the default browser focus outline, so those that use a keyboard to navigate can see where they are focused. None of the buttons are icon-only; every button has a short text label associated with it.

The promo video starts muted and the audio clip uses the standard HTML5 controls, so both can be paused, seeked or have their volume changed from the keyboard.

I tested the button and text colours on the dark control panel (that contains the control buttons) against the Web Content Accessibility Guidelines (WCAG) colour contrast AA standard for normal text using Chrome Developer Tools. The control button labels were the most critical, but all of them passed.

Architecture (MVC)

I attempted to keep the JavaScript separation to the MVC pattern primarily for my own sanity while developing, but it also serves to aid with communication relating to the system.

Model — data/products.json (plus a few state variables at the top of app.js). The products.json file contains the metadata for the products, and the variables keep track of which product is currently being displayed.

View — two parts. scene.js controls everything shown in 3D (the Three.js renderer, camera, lights, and meshes). app.js deals with any updates in the DOM (text in the info panel, highlighted gallery, media tabs, etc).

Controller — the controller is simply the app.js event handler. It provides a central location for handling mouse clicks, key presses, and submits of forms, and calls the necessary methods in the store or Viewer for any actions to be taken.

On the server side, I use api/feedback.php, which reads and writes a SQLite database file and acts as a simple guestbook on the About page only. It is not a full backend.

Statement of Originality

I did not download any of the geometry for any items produced in this project. Each vertex was produced using Three.js LatheGeometry and BufferGeometry within JavaScript, and there are no .glb, .fbx, or .obj files in the repository.

Also, all labels are produced in the browser instead of using an image. Labels are painted, during runtime, on to an HTML canvas and passed to Three.js as a CanvasTexture.

The brand "NovaSoda", the logo, the strapline "taste the orbit," and all of the stories for the products are all fictitious and were created by me purely for the purpose of this coursework. The name and logo are not based on any real company, and I am not using any real company's trademarks.

I utilized a very limited number of well-known third-party libraries, which are included via public CDNs, and I have included credits for each of them on the References page:

  • Three.js r160 — rendering
  • Bootstrap 5.3 — layout & components
  • jQuery 3.7 — DOM helpers and AJAX
  • Google Fonts (Bebas Neue) — bundled locally for display typography

When I used a code snippet with modifications, I included inline comments in the source files to indicate where I originated it and what modifications I made.
